# Graph RAG 如何改造个人知识管理：从线性搜索到图谱推理

> **声明**: 本文基于 Graph RAG 技术前沿分析，结合 KDO 的 Wiki 系统现状，提出将知识图谱检索集成到知识管道的技术路径。Graph RAG 是 2024-2025 年的前沿方向，落地案例尚少。

---

## 一、问题：你的笔记链接在一起，但 AI 看不见

Obsidian/KDO 的 `双向链接` 天然构成了一张知识图谱。但当前 AI（包括 Claude）读取时是**线性扫描文本**——搜索所有包含关键词的文件，返回文本匹配结果。

**这意味着什么？**

```
当前状态（文本 RAG）:
用户问："和 KDO 相关的概念有哪些？"
系统做：  搜索所有 .md 文件中包含 "KDO" 的文本
结果：    找到直接提及 KDO 的页面
漏掉：    通过 Obsidian、AI工作流、知识图谱 间接关联的内容
```

**目标状态（Graph RAG）:**
```
用户问："和 KDO 相关的概念有哪些？"
系统做：
  1. 定位 "KDO" 节点
  2. 遍历图：直接邻居（1-hop）→ 邻居的邻居（2-hop）
  3. 按关系权重排序
结果：  找到直接引用的页面 +
        通过 Obsidian、AI工作流、知识图谱间接关联的内容
```

## 二、Graph RAG vs 传统 RAG：核心差异

| 维度 | 传统 RAG（文本向量） | Graph RAG（知识图谱） |
|------|---------------------|----------------------|
| **检索单位** | 文本块（chunk） | 节点+边（概念+关系） |
| **匹配方式** | 向量相似度 | 图遍历+向量混合 |
| **强项** | 语义相似搜索 | 跨概念关联推理 |
| **弱项** | 漏掉 "语义不相似但逻辑相关" 的内容 | 对叙事性文本提升有限 |
| **适用内容** | 全部文本 | 结构化知识（概念、实体、决策） |
| **维护成本** | 低（自动 chunk） | 中高（需维护链接质量和图索引） |

**关键判断**: Graph RAG 不是为了替代传统 RAG，而是在其上叠加一层图推理。对 KDO 而言，`30_wiki/` 层的概念网络最适合做图索引，`00_inbox/` 和 `10_raw/` 层保持传统文本 RAG 即可。

## 三、对 KDO 的三阶段实施路径

### Phase 1: 链接提取与图构建（P1 优先级）

```
输入:  30_wiki/ 所有 .md 文件
操作:  解析 双向链接 → 构建邻接表 → 导出 GraphML/JSON
输出:  30_wiki/.graph/index.json
```

**节点**: 每个 wiki 页面 = 一个图节点
**边**: `[[页面A]]` 出现在页面B中 = A→B 的有向边
**权重**: 基于链接上下文计算（段落中的引用权重 > 列表中的提及）

### Phase 2: 关系语义化（P2 优先级）

当前 Obsidian 链接只是 "存在关系"，不表达 "什么关系"。引入边语义标签：

```yaml
edges:
  - from: "kdo-protocol"
    to: "graph-rag"
    relation: "requires"        # requires / enables / contradicts / derives-from
    weight: 0.9
```

**关系类型**:
- `enables`: A 使 B 成为可能
- `requires`: B 依赖 A
- `contradicts`: A 与 B 存在矛盾
- `derives-from`: B 从 A 派生出

### Phase 3: 混合检索（P2+ 优先级）

```
用户查询 → 向量检索（快速召回） → 图遍历（扩展关联） → 混合排序 → 返回结果
```

先做向量搜索拿到种子节点，再从种子节点出发做 1-2 hop 图遍历，两层叠加排序。

## 四、前置条件与边界

**Graph RAG 有效的前提**:
1. 笔记中的 `链接` 质量足够高，能真实反映概念之间的关系（而非随意添加）
2. 笔记量 ≥ 100 页（少于这个量全文搜索已经足够）
3. 内容以结构化知识为主（概念、实体、决策），而非叙事性文本

**不适合的场景**:
- 随笔、日记、情感记录类内容
- 链接是机械性添加（如模板中的固定链接）而非语义性关联
- 频繁变动的内容（图索引需要持续重建）

## 五、Call to Action

1. **评估你的链接质量**: 随机抽查 20 条 `链接`，看其中有多少是 "有语义价值的关联" vs "随意添加" vs "模板固定链接"。如果语义价值链接占比 < 50%，先提升链接质量再建图。
