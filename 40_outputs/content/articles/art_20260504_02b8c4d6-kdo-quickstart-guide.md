# Knowledge Delivery OS 快速上手指南：把散落知识变成可交付资产

## Core Thesis

信息过载不是问题，信息无法变成可用的东西才是问题。你收藏了 200 篇文章、写了 50 篇笔记、参加了 10 个课程——然后呢？它们躺在硬盘里，对实际工作的帮助几乎为零。

Knowledge Delivery OS (KDO) 的设计动机就来自这个"然后呢"。它不是一个笔记应用，不是一个知识库，而是一条从原材料到可交付资产的流水线。核心逻辑只有一条：任何输入（飞书文档、网页、AI 对话、课程转录）都应该能变成三种产出之一——一篇可发布的文章、一段可复用的代码、一个可调用的能力。这个从"输入"到"产出"的完整链路，是 KDO 与其他知识管理工具最根本的区别。

## Key Takeaways

1. **KDO 是流水线，不是文件夹**：9 步闭环（捕获 → 注册 → 编译 → 路由 → 生产 → 验证 → 交付 → 反馈 → 改进）定义了知识从原材料到成品的完整路径。每一步有对应的 CLI 命令、目录映射和质量标准。
2. **三层产物，一条溯源链**：内容（文章/报告）、代码（脚本/模板）、能力（Skill/Agent）三种产出共享同一套 source → wiki → artifact 的溯源机制——任何一条交付物都可以追溯到它的原始来源和中间知识加工过程。
3. **本地优先，文件即数据**：所有数据存储为 Markdown + YAML frontmatter，状态管理通过 `state.json`。不依赖云端服务，Git 直接可用作版本管理和协作。
4. **CLI 是刻意设计，不是临时方案**：12 个核心命令覆盖完整生命周期。没有 GUI——目标用户是愿意用命令行的知识工作者。这不是傲慢，是取舍：CLI 的精确性和可脚本化，对知识流水线来说比可视化更重要。
5. **飞书是输入桥，不是替代品**：KDO 不是要取代飞书、Notion——它是要让你在这些工具里沉淀的知识"活过来"，变成可以进一步加工和交付的半成品。

## Draft

### 为什么你需要一条知识流水线

大多数人的知识管理还停留在"收集+分类"阶段。把好东西存进印象笔记、Notion、Obsidian，打好标签，偶尔翻一翻——这最多叫"信息管理"，不是"知识管理"。真正的知识管理应该完成一个闭环：外部的信息进来，经过你的理解和加工，变成可以交付给别人使用的东西。

KDO 把这个闭环工程化了。它的 9 步流程不是凭空设计的——每一步对应真实知识工作中的一种认知操作：捕获是"我发现了一个好东西"、编译是"我得把它消化成自己的理解"、生产是"我要写出一个别人能看懂的东西"、验证是"写得对不对"、交付是"发出去让人看到"、反馈是"别人怎么说"、改进是"下次怎么更好"。

这些操作每个人做知识工作时都在做，只是大部分人没有意识到它们是一个连续的流水线。KDO 的价值就是把隐性的认知流程显性化、标准化，让每一步变得可追踪、可复查。

### KDO 适合谁

如果你符合以下特征，KDO 可能适合你：
- 你的工作产出主要是文字（报告、文章、文档、教程）
- 你有多个信息输入渠道（网页、课程、对话、文档）需要整合
- 你需要在知识和最终交付物之间建立明确的溯源关系
- 你不介意使用命令行

如果你需要多人实时协作编辑、主要处理的是图片/视频等非文本知识、或者团队中非技术人员占多数——KDO 当前版本不是最好的选择。但即使在这些场景下，KDO 的知识流水线思路仍然可以作为参考。

### 从飞书到 KDO：一个典型工作流

假设你在飞书文档里写了一篇行业分析。传统做法是：写完、分享链接、结束。KDO 的做法是：

1. `kdo capture` 把飞书文档拉到 inbox
2. `kdo ingest` 把原始文档编译为结构化的 wiki 页面
3. `kdo enrich` 对 wiki 页面进行三步编译（浓缩→批判→对标），把"原始信息"升级为"可复用的知识卡片"
4. `kdo produce` 将知识卡片生成为面向特定读者的文章初稿
5. `kdo validate` 检查初稿的质量（骨架完整性、引用一致性等）
6. `kdo ship` 记录交付信息，把状态从未完成变为已完成
7. `kdo feedback` 收集读者反馈
8. `kdo improve` 将反馈转化为具体的改进动作

每一步都是可选的——你不需要每次都用全部 9 步。但这个流程的存在意味着：当你想要做完整流程时，基础设施已经就位。

### 与 Obsidian 的关系

KDO 和 Obsidian 是互补关系，不是竞争关系。Obsidian 负责知识图谱的可视化和日常笔记编辑，KDO 负责知识的结构化加工和交付流水线。二者共享 Markdown + frontmatter 的内容格式，可以在同一个 vault 中共存——Obsidian 是你在知识花园里漫步的界面，KDO 是你把花园里的草药制成成药的工厂。
