# AI 集群深度调研方法论：认知三角测量与四级置信度体系

> **声明**: 本文基于 Kimi Deep-Research-Swarm 方法论提炼核心框架，分析了多智能体并行调研的设计原理。置信度判断来自方法论原文的设计目标，实际执行效果需独立验证。

## 一、核心创新：不是更快，而是更可靠

Kimi Deep-Research-Swarm 的核心理念是 **认知三角测量（Epistemic Triangulation）**：在不同研究维度上发散探索 → 检测重叠与矛盾 → 深度验证 → 收敛为经过交叉验证的综合结论。

这与传统 "一个 Agent 线性搜索" 的调研方法有本质区别：

| 维度 | 传统 AI 调研 | Kimi 集群调研 |
|------|-------------|---------------|
| Agent 数量 | 1 | ≥10 并行 |
| 搜索预算 | ~10-20 次 | ≥200 次（Route B） |
| 质量保障 | 单点来源 | 四级置信度 + 跨 Agent 交叉验证 |
| 冲突处理 | 隐式/忽略 | 显式记录，从不压制 |
| 输出粒度 | 笼统总结 | 每条发现含 Claim/Source/URL/Date/Excerpt/Context/Confidence 七字段 |

**关键设计决策**：集群并行服务于**认知鲁棒性**，而非仅仅追求速度。

## 二、自适应路由：四种调研模式的自动选择

系统根据任务特征自动选择四条路线之一：

| 路由 | 适用场景 | 搜索预算 | 关键行为 |
|------|----------|----------|----------|
| **Route A** 广域搜索 | 无文件、主题宽泛 | 标准 | 广度优先，覆盖问题空间全貌 |
| **Route B** 聚焦搜索 | 无文件、主题明确、需深度 | ≥200 次 | ≥10 Agent × ≥20 搜索，深度挖掘 |
| **Route C** 纯文件研究 | 有文件、禁止外部搜索 | 文件解析 | 尊重 "不上网搜索" 的用户意图 |
| **Route D** 文件增强 | 有文件、允许补充搜索 | 文件+搜索 | 平衡文件内容与外部补充信息 |

**Route C 的设计体现了对用户约束的尊重**——不是所有调研都需要上网。某些场景下（如内部文档分析、法律合同审查），禁止外部搜索是正确的。

## 三、四级置信度：让不确定性可见

Phase 4 的交叉验证引擎将每条发现标注为四个等级：

| 级别 | 定义 | 触发条件 |
|------|------|----------|
| **高置信度** | 可安全引用 | ≥2 个 Agent 从独立来源确认同一发现 |
| **中等置信度** | 可引用但需标注 | 1 个 Agent 从权威来源确认 |
| **低置信度** | 需进一步验证 | 来源薄弱或单一未验证 |
| **冲突区域** | 存在分歧 | Agent 间统计分歧或解释分歧 |

**冲突从不压制**——它们是最高价值的信号，指向 "行业认知尚未收敛" 的领域。

## 四、对 KDO 的可迁移价值

这套方法论对 KDO 的 Enrich 阶段有直接启发：

1. **维度分解**：KDO 的 Enrich 阶段当前做单一视角的 "浓缩/质疑/对标"。可以引入 Kimi 的 ≥10 维度分解（技术图景、市场格局、监管政策、竞争动态、用户视角、供应链、地理差异、历史演进、新兴颠覆、利益相关者），让每篇 wiki 页面的 Enrich 更立体。

4. **Route A vs Route B 的搜索预算分档**：KDO 的 Ingest 阶段可以根据源文件大小和主题复杂度，自动选择 "快速入站"（类似 Route A）或 "深度入站"（类似 Route B）。

## 五、风险与局限

| 风险 | 说明 |
|------|------|
| **执行成本** | Route B 的 ≥200 次搜索 + ≥10 Agent 并行，单次调研 API 费用可能 $5-20 |
| **幻觉一致性** | 多个 Agent 可能基于相同的错误来源得出相同的错误结论，交叉验证无法检测 |
| **编排复杂度** | 多 Agent 的上下文传递损耗、重复搜索、结果去重需要强大的编排基础设施 |
| **对普通用户的可用性** | 完整执行 Kimi 方法论需要技术团队支持，非技术用户难以独立操作 |
