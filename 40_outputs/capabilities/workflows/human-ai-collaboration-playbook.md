# 人机协作操作手册

> 基于吴恩达提示词工程框架，为 KDO 工作流中的所有 agent 和人类用户提供统一的人机协作规范。

## 角色定义：AI 是合伙人，不是工具

| 维度 | 旧心智模型 | 新心智模型 |
|------|-----------|-----------|
| AI 定位 | 问答机器 / 信息检索工具 | 需要被校准、追问、约束、反驳的智能合作者 |
| 协作方式 | 丢问题 → 拿答案 → 交差 | 喂背景 → 给约束 → 多轮迭代 → 交叉验证 → 闭环复盘 |
| 价值来源 | AI 省事（写邮件、做纪要） | AI 拔高上限（决策质量从 65→75，强项从 85→95） |
| 提示词 | 学习模板和咒语 | 提供独特的业务上下文和约束 |
| 输出质量 | 取决于提示词模板质量 | 取决于独特输入 × 迭代深度 × 模型能力 |

## 关键原则

### 1. 守脑如玉
AI 越来越聪明时，人必须保持大脑锋利。AI 省掉的那些事（写邮件、起标题）本来就不值钱。AI 真正的价值是拔高上限。

**操作检查**：
- [ ] 这个问题我是否已经想清楚了？还是指望 AI 替我想？
- [ ] AI 给的这个答案，我有能力判断它对不对、好不好吗？
- [ ] 如果我没有判断力，先退回去把问题拆解到我可判断的子问题级别

### 2. 拔上限非填下限
不要用 AI 来补你的短板——AI 补短板只能达到 60-70 分的平均水平。用 AI 来拉长你的长板——把你本来就强的能力从 85 分拔到 95 分。

**操作检查**：
- [ ] AI 这次的输出是帮我省了时间（填下限）还是在某个方向打开了我的视野（拔上限）？
- [ ] 如果只是省时间，这个输出的质量是否足够？还是只是看起来完整但内容空洞？

### 3. 迭代优于模板
每个公司面对的问题独一无二——抄别人的提示词等于穿别人的鞋跑自己的路。好的提示词在对话中长出来。

**操作检查**：
- [ ] 我是否完整喂入了我的独特背景和约束？还是只用了一句话提问？
- [ ] 我是否让 AI 先复述了它的理解，以确认没有偏差？
- [ ] 我是否要求多种方案而非唯一答案？

### 4. 反谄媚是每轮交互的必须步骤
AI 天生迎合使用者。不要用正向形容词（"这个方案是不是很有前景？"）、先问缺点再问优点、引入反对者角色。

**操作检查**：
- [ ] 我的提示词中是否有"积极""优质""庞大""有潜力"这类正向形容词？
- [ ] 我是否先让 AI 指出缺点和风险，再让它说优点？
- [ ] 对重要决策，我是否引入了至少 2 个反对者角色做交叉验证？

### 5. 信息要充分，但不超过相关度边界
没有背景 AI 只能给通用行货。但无关信息会严重干扰输出。

**操作检查**：
- [ ] 我给的背景信息是否都和当前问题直接相关？
- [ ] 不同问题是否用了不同的对话窗口（避免交叉污染）？
- [ ] 我是否标注了不确定的信息为"待确认"？

## 场景速查

### 场景 A：写作
```
1. 先磨大纲（出 3 版：激进/稳健/反常规 → 选最优合并）
2. 大纲确认后，逐段展开要点（不一次性生成全文）
3. 要点确认后，展开全文（你的工作是删空洞句子，不是改）
4. 润色逐段逐句（永远不说"帮忙润色全文"）
5. 每段润色指定：段落功能 | 当前不足 | 保持风格 | 改进1-2个具体点
```

### 场景 B：判断/决策
```
1. 制定评分标准（可证伪的维度）
2. 分维度逐一打分（禁止先给总分）
3. 多模型交叉验证（用当下最强模型）
4. AI 辅助判断但高风险决策绝不只依赖 AI（法律/财务/医疗/投资）
5. 设计证伪实验而非证实实验："哪些证据出现时说明这个方向不值得继续？"
```

### 场景 C：学习/内化
```
四遍学习法：
1. 手写笔记（强制理解——决定"什么值得写、怎么写才准确"）
2. 口述展开（发现讲不清楚的地方就是没真懂）
3. AI 整理（让 AI 帮你整理、补充、指出遗漏）
4. 分享讲授（教给别人，完成知识闭环）

注意：不让 AI 代替第 1 步和第 2 步——那会让整个学习建立在 AI 的理解之上
```

### 场景 D：头脑风暴（AI 杀手应用）
```
1. 先写独特资源（你有什么别人没有的？）
2. 再写独特约束（钱/团队/时间/政策等硬约束）
3. 要求从 5 个不同方向各出 1 个方案
4. 每个方向深挖时追问：最关键假设是什么？如果假设不成立？
5. 让 AI 给自己的每个方案打分并说明最可能失败的原因
6. 只深挖 AI 自己能指出致命缺陷的方案
```

### 场景 E：日常问题
```
简单任务（改文案、起标题）：
- 用快速模型
- 一次问清楚 + 少量迭代
- 不需要完整的 7 层工作流

复杂任务（商业模式判断、跨领域推理、长上下文分析）：
- 用当下最强模型
- 完整 7 层工作流
- 至少 10 轮迭代
- 必须经过反谄媚过滤
```

## 失败模式速查

| 症状 | 原因 | 对策 |
|------|------|------|
| AI 一直在说好话 | 提示词含正向形容词 | 删词 + 先问缺点 |
| 方案漂亮但做不了 | 没给约束 | 补充独特约束 |
| 越问越浅 | 你没有判断力 | 把问题拆到你能力范围内 |
| 上下文太多 AI 乱了 | 无关信息污染 | 新开窗口，只给相关信息 |
| 输出空洞没密度 | 直接生成全文 | 退回大纲→要点→全文流程 |
