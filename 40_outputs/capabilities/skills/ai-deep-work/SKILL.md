# AI 深度工作法（AI Deep Work）

> 将吴恩达提示词工程框架编码为可执行的 agent / 人类工作流。触发条件：复杂推理、多轮迭代思考、需要 AI 协作的开放式问题。

## 触发条件

当 agent 或人类用户面对以下场景时激活本 skill：
- 一个未完全定义的复杂问题，需要 AI 协作探索
- 需要多个视角交叉验证的决策
- 需要从模糊想法收敛到可执行方案的创意过程
- 用户已有一定背景知识但不知道如何高效与 AI 协作

## 输入

1. **问题描述**：一句话定义你要解决的问题（越具体越好）
2. **已知上下文**：你已经知道的业务背景、约束、历史尝试、失败原因
3. **期望产出**：你想要什么形式的输出（方案列表/分析报告/行动计划）

## 工作流（7步循环）

### Step 1: 背景层 — 把已知信息完整喂入
```
操作：先穷尽你已知的所有背景。不要问AI"你需要什么信息"，先写你能写的。
内容要素：业务阶段 | 目标用户 | 产品形态 | 价格区间 | 已有资源 | 历史尝试 | 失败原因 | 当前约束 | 不可接受方案 | 时间窗口 | 验证指标
指令模板：
"以下是我的完整业务背景：[背景段落]。请先复述你对我的业务背景、关键约束和当前问题的理解。
然后告诉我，你还缺少哪些关键信息，才能给出更有针对性的建议。"
```

### Step 2: 选项层 — 生成多个不同方向的方案
```
操作：要求至少 5 个不同方向的方案，不要唯一答案。
指令模板：
"基于[具体约束]，请从以下 5 个不同方向给我各出 1 个方案：
(1) 零成本方向 (2) 杠杆他人资源的方向 (3) 极速验证方向（3天内出结果）
(4) 反常识方向（同行都不这么做但可能有效）(5) 你最不看好的方向但告诉我为什么它可能成功"
失败模式：只生成了 1 个方案就开始深挖 → 先发散再收敛
```

### Step 3: 假设层 — 拆解每个方案的关键假设
```
操作：要求 AI 说明每个方案成立的关键假设。
指令模板：
"请逐一拆解方案[编号]：它依赖的 3-5 个关键假设是什么？
如果不做任何验证，哪些假设最可能不成立？"
```

### Step 4: 反驳层 — 多角色交叉质疑
```
操作：新开窗口（避免上下文交叉污染），让 AI 扮演不同反对者角色。
角色清单：
- 投资人视角："你是[赛道]投资人，十年经验。如果你决定不投，因为什么？"
- 竞品视角："你是[竞品]的市场负责人。这个方案会威胁到你吗？为什么不会？"
- 失败复盘者："你是一年后复盘这个项目的创始人。最可能的 3 个失败原因？"
- 客户视角："你是[目标用户画像]。这个方案对你来说最大的顾虑是什么？"
- 运营视角："你是 COO。这个方案落地时最大的执行困难是什么？"
失败模式：角色描述太泛（"你是投资人"）→ 角色越具体，反驳越有力量
```

### Step 5: 标准层 — 分维度打分（先分项后总分）
```
操作：先制定评分标准，再分维度打分，最后汇总。不做"直接打分"。
指令模板：
"请为这个方案制定 5 个评分维度，每个维度配一个可证伪的评分标准。
然后按维度逐一打分，最后汇总。禁止先给总分。"
失败模式：AI 先给总分 → 后续分析都在为这个总分辩护
```

### Step 6: 实验层 — 转化为最小验证行动
```
操作：将结论转化为可在 7 天内执行的最小验证步骤。
指令模板：
"基于当前最优方案，设计一个 7 天内可完成的最小验证实验。
包含：实验目的 | 成功标准 | 操作步骤 | 所需资源 | 风险预案"
```

### Step 7: 复盘层 — 把结果重新喂给 AI
```
操作：实验结果出来后，重新喂给 AI 做分析和下一步决策。
指令模板：
"以上是[实验名称]的结果：[结果描述]。
请分析：哪些假设得到了验证？哪些被推翻了？下一步应该做什么？"
```

## 失败模式速查

| 症状 | 原因 | 对策 |
|------|------|------|
| 每轮 AI 都给你新方向，但每个都浅 | 你没有判断力给出有效反馈 | 回退到 Step 3，先把问题拆到你能判断的子问题级别 |
| AI 一直在说好话 | 你的提示词里有正向形容词 | 删掉所有"积极""优质""有潜力"等词 |
| 方案看起来都对但做不了 | 你没给约束 | 回到 Step 1，补充独特约束 |
| 连续 3 轮改进都是微调措辞 | 模型能力触顶 | 换更强模型，不要继续在提示词上用力 |
| 上下文太多 AI 反而乱了 | 无关信息污染 | 新开窗口，只给当前问题相关的背景 |

## 关键原则

1. **迭代优于模板**：好的输出在对话中长出来，不是一次性设计出来
2. **反谄媚是每轮迭代的必须步骤**：尤其是 Step 4，不跳
3. **模型能力 × 提示词能力是乘法**：复杂问题用最强模型
4. **AI 拔上限不填下限**：你是决策者，AI 是选项生成器和质疑者
