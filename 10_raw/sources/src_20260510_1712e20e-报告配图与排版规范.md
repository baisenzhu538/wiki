# 报告配图与排版规范

用于 Step 9：生成专业报告的视觉规范。完整视觉规范见 `style-guide.md`，本文聚焦配图/数据图/章节标题。

---

## 🔴 BLOCKING 规则：Takeaway Title

**所有章节标题、图表标题必须是结论句**（含动词、含数字/方向），不能是话题词。

| ❌ 话题式（禁用） | ✅ 结论式（必用） |
|----------|----------|
| ## Q3 营收数据 | ## Q3 营收同比增长 41%，但同店增速仅 8% |
| ## 竞品对比 | ## 瑞幸在 6 个维度领先，客单价落后星巴克 35% |
| ## 市场规模 | ## 中国现制咖啡 2024 年达 1,500 亿，CAGR 24% |
| ## 风险分析 | ## 核心风险为加盟商满意度仅 45%，低于行业 15pp |
| *图 1：营收数据* | *图 1：Q3 营收同比 ↑41%，但新店贡献占比达 62%* |

**公式**：`[主体] + [动词] + [数字/方向] + [关键限定]`

**硬规则**：
- 标题 ≤ 20 字（中文）/ 15 词（英文）
- 必须含至少 1 个数字或方向词（↑/↓/领先/落后/超过/低于/达/仅）
- 读者仅看所有 H2 标题 + 执行摘要应能把握全文结论
- **Step 9.5 机械校验**：所有 H2 标题必须匹配正则 `\d|↑|↓|领先|落后|超过|低于|达|仅|增|降`

## 配图策略（重要区分）

商业调研报告 **数据图 > 概念图**。严格区分两类：

| 类型 | 工具 | 占比 | 定位 |
|------|------|------|------|
| **📊 数据图**（必须）| Mermaid / 表格 / 真实数据图表 / 屏幕截图 | ≥70% | 传递事实与数据 |
| **🎨 概念图**（可选）| AI 生成（minimal 风格）| ≤30% | 封面、章节过渡、隐喻 |

> ⚠️ **常见翻车**：把 AI 生成的 "库存图" 作为主力配图会让报告显得不专业。数据图才是商业报告的灵魂。

### 数据图生成方式

| 场景 | 方式 |
|------|------|
| 趋势折线、条形图 | Mermaid 或 Markdown 表格 |
| 竞争格局矩阵 | Markdown 表格（二维）或 Mermaid mindmap |
| 产业链流程 | Mermaid flowchart |
| 甘特图/时间线 | Mermaid gantt |
| 数据仪表盘 | Markdown 表格 + 粗体数字高亮 |
| 复杂可视化 | 输出数据，调用 `/text-visualizer` 生成 |
| 真实截图 | 官网/App/后台截图（插入 images/）|

### 概念图生成（仅封面和章节过渡）

```
1. 仅为封面、章节过渡、执行摘要后插入概念图
2. 风格固定：minimal（极简商业风）
3. 提示词 50-120 词，英文
4. 颜色用自然语言描述，禁止 hex
5. 保存到 images/，同时把提示词存到 prompts/
```

**降级方案**：无图像生成工具时，用 `<!-- 配图位置：[描述] -->` 占位，提示用户用 `/article-illustrator` 补充。

## 图表硬约束（基于 Storytelling with Data + Tufte）

### 图表选型表（硬约束）

| 数据类型 | ✅ 用 | ❌ 禁用 |
|---------|------|--------|
| 时间趋势 | 折线图 | 饼图、3D |
| 类别对比 | 条形图（从 0 开始，条宽 > 间距）| 双 Y 轴、3D |
| 部分对整体 | 100% 堆叠条（≤5 类）| 饼图（>5 类）、3D 饼 |
| 多维对比（≤5 维）| 雷达图 | 表格（有更好选择）|
| 多维对比（>5 维）| 热力矩阵 | 雷达图 |
| 排名/大小 | 横向条形图 | 折线图 |
| 流程/阶段 | Sankey / 漏斗图 / 流程图 | 纯文字 |
| 关系/依赖 | 网络图 | 表格 |

### "灰色是主角" 原则

- **默认全图灰阶**（`#595959`、`#BFBFBF`）
- 仅对需强调的一条数据用主色 `#005EB8`
- 禁用 Excel 默认彩虹色
- 警示色仅限风险/趋势：红 `#D32F2F`（下降/风险）、绿 `#2E7D32`（上升/积极）

### 坐标轴与结构

- **Y 轴必须从 0 开始**（除非是变化率，且必须明确标注截断）
- 时间轴间距一致
- 坐标轴标题必须带单位：`营收（亿元）`
- 删除：网格线、边框、3D 效果、冗余图例
- 数据标签直接贴在柱/线末端

### 每张图底部必备

```
Source: [来源名称] | [链接] | [获取日期] | N=[样本量] | 注：[特殊说明]
```

## 配图数量参考（仅作下限）

| 报告类型 | 字数 | 数据图（必须）| 概念图（可选）| 总计 |
|---------|------|--------|--------|------|
| 调研简报 | 1000-2000 | 2-3 | 0-1 | 2-4 |
| 企业情报档案 | 3000-5000 | 4-6 | 1-2 | 5-8 |
| 竞品分析报告 | 4000-6000 | 5-7 | 1-3 | 6-10 |
| 行业调研报告 | 5000-8000 | 6-9 | 2-3 | 8-12 |
| 项目评估报告 | 3000-5000 | 4-6 | 1-2 | 5-8 |

**必配图位置**（不可省略）：
1. 封面/题图（概念图，全文主题）
2. 竞争格局/市场格局图（数据图：矩阵或象限）
3. 核心数据仪表盘（数据图：3 个大数字）
4. 假设验证结果（数据图：状态表 ✅❌⚠️🔄）
5. 决策建议（数据图：表格或决策树）

## 排版节奏

```
# 报告标题（Takeaway Title 格式，仅 1 个 H1）

[封面概念图]

> 📊 **执行摘要**
> [SCQA + 3 KPI 仪表盘]

---

## 一、[章节 Takeaway Title——必须含数字/方向]

[1-2 句导语——引出本章要回答的问题]

### 1.1 [小节 Takeaway Title]

正文段落...

![图 1：Takeaway Title](./images/fig1.png)
*Source: [来源] | [日期] | N=[样本] | 注：[说明]*

> 💡 **关键洞察**：[非显而易见的结论，1-2 句]

**小结**：[3 行内要点提炼]

---

## 二、[章节 Takeaway Title]
...

---

## 盲区声明
## 参考资料（L1-L2 权威 | L3-L5 次要 分组）
## 延伸操作
```

### 排版原则

1. **引用块**（`>`）：执行摘要（📊）、关键洞察（💡）、风险提示（⚠️）、专家引言（"）
2. **分隔线**（`---`）：章节之间，不要用 `===` 或过度装饰
3. **表格**：结构化对比数据（见 `style-guide.md` §5）
4. **粗体**：关键结论、核心数据
5. **图注**：斜体 `*Source: ... | ...*`
6. **每章开头**：1-2 句导语
7. **每章结尾**：粗体 "**小结**：" 引导的 3 行内要点

---

## Anti-Slop 禁用词（Step 9.5 机械校验）

**禁用**：
- 英文：Smart, Comprehensive, Seamless, Innovate, Leverage, Holistic, Cutting-edge, Best-in-class
- 中文：全方位、一站式、赋能、颠覆、引领、打造、共创、生态、矩阵化、体系化（空泛时）

**替换为具体证据陈述**：
- ❌ "该公司具有全方位的竞争优势"
- ✅ "基于 3 个独立信源：该公司在 A/B 维度领先竞品 30%、20%，但在 C 维度落后 15%"
- ❌ "市场前景广阔"
- ✅ "市场规模 Top-Down 估算 520 亿，Bottom-Up 估算 480 亿，收敛度 8%，可信度 ✅高"
- ❌ "团队实力雄厚"
- ✅ "创始人有 3 次连续创业经验（2 次成功退出）；核心团队 5 人中 4 人来自头部公司"
