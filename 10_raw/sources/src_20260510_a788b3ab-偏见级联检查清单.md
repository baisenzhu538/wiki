# 偏见级联检查清单

在 Pre-Mortem 之后执行。偏见会叠加放大，每种偏见的存在都可能触发下一种。

## 五大偏见检查

| 偏见类型 | 检查问题 | 补救方法 |
|---------|---------|---------|
| **确认偏误** | 我是否只找了支持假设的证据？ | 主动搜索反面证据、失败案例 |
| **锚定偏误** | 我是否被第一个看到的数据锚定了？ | 忽略第一个数据，独立估算后再比较 |
| **幸存者偏误** | 我是否只看了成功案例？失败的呢？ | 专门搜索"失败案例""退出市场""倒闭" |
| **光环效应** | 我是否因为品牌/创始人光环而高估了？ | 只看数据，不看故事；单独审查每项指标 |
| **可得性偏误** | 我是否因为某信息容易获取就过度依赖？ | 检查信息来源的多样性，主动寻找难以获取的数据 |

## 典型偏见级联路径

```
锚定在某个市场规模数字
    ↓
只找支持该数字的报告（确认偏误）
    ↓
只看该市场中的成功案例（幸存者偏误）
    ↓
因创始人光环高估执行能力（光环效应）
    ↓
结论："市场很好，团队很强"
```

实际上每一步都在放大误差。

## Pre-Mortem 执行模板

Gary Klein 开发，研究表明可将风险预测准确率提升 30%。

```
"假设六个月后证明我的结论是错的——
  1. 最可能错在哪里？
  2. 我忽略了什么信息？
  3. 我的哪个假设其实最脆弱？
  4. 有没有一个完全相反的解释也能说通？"
```

## Devil's Advocacy（红队审视）

在得出结论后，刻意花 5 分钟为**相反结论**构建最强论据：

```
原始结论：[我的结论]
最强反驳：[如果结论是错的，最有力的反对论据是什么？]
→ 如果反驳论据同样有力：降低原结论置信度，或标注为"争议性结论"
→ 如果反驳论据明显较弱：维持原结论，但在报告中提及该反驳并说明为何不采信
```

## 信息冲突裁决规则（配合信源分级使用）

1. **隔离陈列**：并列展示矛盾双方的评分与依据
2. **强制标注**：当两方信源分差 < 1.5 时，必须同时陈述两种可能性
3. **动机分析**：结合利益冲突因子，推导信息发布的潜在动机

示例：
- 竞品 CEO 声称"用户增长 200%"（信源分 3-1=2，存在利益冲突）
- 第三方数据平台显示增长 80%（信源分 4+1=5，时效高）
- 结论：采信后者（5 > 2），前者标注为"当事人声明，与第三方数据存在分歧（差距 2.5 倍）"
