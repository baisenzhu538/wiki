# 低成本测试MVP：用实验思维10倍速验证假设

> 落地工具箱卡片（磨设计）。[[yt-model-pan-product-execution-toolkit]] | [[yt-model-pan-product-36-strategies]] | [[一堂]]

## Summary

低成本测试MVP提供从"不做产品"到"全量开发"的六档武器库——验证成本从左到右越来越高，核心原则是用最低成本学到最关键认知。不动开发 → 模拟产品 → 提供产品 → 极小开发 → 正式开发 → 全量开发。

## [Condense]

### MVP设计武器库

| 层级 | 方式 | 成本 |
|------|------|:--:|
| 不做产品 | 直接验证/提供VIP服务但用现成工具 | Min |
| 模拟产品 | 尽量用人替代 | ↑ |
| 提供产品 | 最小成本憋大招 | ↑ |
| 极小开发 | 核心功能仅满足验证需求 | ↑ |
| 正式开发 | 按产品标准开发 | ↑ |
| 全量开发 | 完整产品全量投入 | Max |

> 适合场景：用较低的成本大幅降低系统性风险
> 进步方式：学一堂课「精益」系列课程

## Visual Analysis

卡片将MVP武器库从左到右排列——成本依次递增，形成一条"成本光谱"。最左侧"不做产品"被标红或强调——暗示最容易被忽视的选项是"不做产品也能验证"。从左到右的箭头暗示存在"最佳停靠点"——在验证目标达到后就应该停止向右移动，而不是惯性推进到全量开发。
