# 智能体架构：PEAS与五层Agent模型

> 来源：Russell & Norvig《Artificial Intelligence: A Modern Approach》第4版，经AI思维卡深度加工。

## Summary

PEAS + 五层Agent架构是描述任何决策系统的通用语言。PEAS（Performance/Environment/Actuators/Sensors）把模糊目标拆成四个可设计的工程槽位；五层架构（Simple Reflex → Model-based → Goal-based → Utility-based → Learning Agent）提供一张诊断"这个系统卡在哪一层"的地图。核心主张：**升级的本质是换架构，不是加算法**——拖延不是性格问题，是reflex agent在打multi-agent动态战。

## [Condense]

### PEAS：描述任何任务环境的四元组

| 字母 | 工程含义 | 自我反思用法 |
|------|---------|------------|
| **P**erformance | 性能度量 | 成功长什么样？谁打分？ |
| **E**nvironment | 环境 | 在哪儿玩？谁是其他玩家？ |
| **A**ctuators | 执行器 | 能调用的动作集是什么？ |
| **S**ensors | 传感器 | 能拿到什么反馈？盲区在哪？ |

PEAS的核心价值不是"把事写清楚"，而是**暴露空槽位**。90%的OKR失败是因为只填了P（目标数字），漏了E（市场环境）、A（可用手段）、S（反馈机制）——你定了"年度营收X亿"但完全没说在什么市场、用什么手段、怎么拿反馈。这不是没努力，是被设计成失败。

### 任务环境的7个维度

完全可观察 vs 部分可观察 / 确定性 vs 随机 / 离散 vs 连续 / 静态 vs 动态 / 单agent vs 多agent / 已知 vs 未知 / 回合制 vs 序列。

关键洞察：学校是"完全可观察+确定性+离散+静态+回合制"——约等于温室。真实工作环境是"部分可观察+随机+连续+动态+多agent+序列"——所有hard模式叠满。**从学校到职场让人怀疑人生，是因为你被训练成下棋手，结果被扔进了战场**。

### 五层Agent架构：一张自我诊断地图

```
Simple Reflex           "看到刺激就反应"              最低行为层级
Model-based Reflex      "脑内有世界模型再反应"         
Goal-based              "为达成目标做规划"             
Utility-based           "在多个目标间权衡取舍"         
Learning Agent          "持续更新自己的某个部件"       最高行为层级
```

每一层的关键跃迁：

1. **Simple Reflex**：if-then规则。看到脏就扫。没有记忆，没有规划。大多数人的默认模式。
2. **Model-based Reflex**：维护一个"世界现在是什么状态"的内部模型。知道传感器看不到的地方可能发生了什么。
3. **Goal-based**：有了目标状态，能反向规划"从当前状态到目标状态需要哪些动作"。
4. **Utility-based**：当多个目标冲突时，用效用函数做trade-off。不只是"达成目标"，而是"在约束下做到最好"。
5. **Learning Agent**：有一个明确的**被更新的部件**——要么世界模型、要么效用函数、要么动作策略、要么评分体系。指不出"我更新了哪个部件"的学习，就是消费内容。

### 理性 = 在已知信息下最大化预期效用

四条核心约束：
- 不要求结果最优，只要求**决策最优**（区分决策质量和结果质量）
- 信息有限是默认设置（你不是先知）
- 效用要先定义清楚（这就是P）
- "更努力" ≠ "更理性"，**重新定义性能度量**才是

**应用**：抱怨环境（E）是徒劳——E是给定的。你能动的只有S（拿更好的信息）和A（练更精的动作）。

### 学习 = 让Agent的某个部件随经验更新

真正的learning agent能回答："读完这本书/上完这门课，我更新了哪个部件？"——世界模型里的某个节点被修正了？效用函数里某条权重被调整了？动作策略库新增了一个pattern？评分体系里校准了某个阈值？

答不出 = 你只是消费了内容，不是学习了。

## [Critique]

### 1. PEAS在P模糊领域失效

PEAS默认P可定义。但人生最难的问题恰恰是P没人能给——养育、亲密关系、艺术、伦理。强行PEAS化会把丰富目标退化为可测但失真的代理指标（古德哈特定律）。在这些领域，应先做"P探索agent"（目标：找到正确的P），再做执行agent。

### 2. 五层架构不完全适用于神经网络时代

现实中的LLM是混合体——同时包含reflex（模式匹配）、model-based（世界知识）、goal-based（遵循指令）元素，不完全落在单一层级。五层架构作为思维工具仍然有效，但不应教条化。

### 3. 贝叶斯偏向

AIMA全书倾向概率推断，对非单调推理、符号主义、模糊逻辑处理得相当敷衍。Norvig本人是公开的贝叶斯主义者。不要把"效用可累加为标量"当成唯一真理——人脑不天然按效用累加运作（卡尼曼），且很多决策面对的是奈特式不确定性（连概率分布都未知）。

### 4. 源材料限制

本卡片基于AI思维卡（认知升级系统v3.2输出）加工，而非AIMA原书直接萃取。PEAS和五层架构的细节经过两层转译（原书 → 个人阅读笔记 → 本卡片），可能存在精度损失。获取原书逐字稿后可进一步校准。

### 5. 外部攻击：Gary Klein的识别启动决策（RPD）

Gary Klein（认知心理学家，Macrocognition学派创始人，著有《Sources of Power: How People Make Decisions》）通过消防指挥官、军事指挥员、ICU护士的实地研究得出与AIMA理性模型直接冲突的结论：**专家在真实时间压力下不使用效用最大化**。Klein提出的"识别启动决策"模型（Recognition-Primed Decision, RPD）描述了专家的实际决策路径：看到情境特征 → 匹配经验库中的原型模式 → 心理模拟验证可行性 → 执行第一个可行的方案（而非比较多个方案的期望效用）。

这不是"不够理性的权宜之计"，而是对奈特式不确定性的自适应最优解——在fireground上，你连概率分布都构建不出来，更别说计算期望效用。Klein与卡尼曼长达三十年的辩论（Naturalistic Decision Making学派 vs Heuristics and Biases学派）的核心分歧在于：**实验室里发现的认知偏差，在真实领域的专家身上是否构成缺陷**。Klein的田野数据显示，经验丰富的消防指挥官在"秒级决策窗口"中做出的直觉判断，事后分析往往优于耗时做效用计算的结果。

对PEAS/Agent架构的冲击：如果专家的最佳决策路径不是"定义P → 计算效用 → 选最大期望值"，而是"识别模式 → 心理模拟 → 执行首个可行解"，那么五层架构隐含的"层级越高越好"的价值判断需要被质疑。RPD不是reflex agent的退化，而是一种AIMA框架未能充分建模的认知架构——它利用压缩的经验模式实现了在极端约束下的高速自适应决策。

### 使用场景

**应该用**：设计自动化系统/工作流、诊断"为什么流程不工作"、决定职业转型或技能投资、多人团队角色澄清。

**不应该用**：

| 场景 | 失效机制 | 替代方案 |
|------|---------|---------|
| P本身在争议中的任务（教育、艺术、亲密关系） | PEAS强制定义P触发古德哈特定律——指标取代目标，系统开始优化可测代理变量而非真实价值。把"好的教育"PEAS化为"考试分数"后，学生会优化分数而非理解，教师会教应试技巧而非思维 | 用Donald Schon的**反思性实践框架**（The Reflective Practitioner, 1983）：行动 → 反思 → 重新框定问题 → 再行动，不预设固定P，让P在每个循环中被重新谈判。具体操作：用"行动后回顾"（After Action Review）替代"KPI设定" |
| 秒级时间窗口的危机决策（火灾现场、急诊分诊、军事交火） | 五层架构默认有"从reflex攀升到utility-based"的计算时间，但秒级危机要求跳过中间层。utility-based决策的计算成本（定义变量→赋权重→算期望值→比较）在这种场景下是危险延迟，而非更优决策 | 用Gary Klein的**RPD模型**（Recognition-Primed Decision）：构建经验模式库（pattern repository）+ 心理模拟验证循环（mental simulation），训练"识别 → 匹配 → 模拟 → 执行"的快速通路。具体操作：用"关键决策法"（Critical Decision Method）访谈提取专家模式，而非写效用函数 |
| 强烈不确定的探索期（噪声反馈、未知未知主导、连变量名都不知道） | Learning Agent依赖反馈信号来更新部件，但噪声反馈会让它学到虚假模式——相当于在随机数据上做梯度下降。在"哪些变量重要"都不知道的阶段，优化是对噪声的过拟合，越学越偏 | 用Dave Snowden的**安全失败探测试验**（Safe-to-fail Experiments, Cynefin框架）：不做优化，做探测——设计最小成本、最小后果的干预，只观察不校准，积累足够信号后再切换为utility-based。具体操作：跑≥5个低成本平行探针（parallel probes），观察≥3个周期后再做第一次模式判断 |

### 一句话烧到骨髓

**智能不是魔法是闭环，理性是数学不是道德，升级的本质是换架构。**

## Action Triggers

以下触发场景设计原则：每个触发器有可验证的成功指标，不依赖"感觉变好"或"更有信心"等主观判断。

### 触发器1：设计新工作流或自动化系统

**触发场景**：要搭建一个新的工作流、自动化管道、或任何决策系统（比如"我要搭一个内容生产流水线"、"我要设计一个客户跟进系统"）。

**第一个动作**：在白板上画PEAS四元组表。按以下顺序填写——先填A（执行器/可用工具清单）和S（传感器/反馈来源清单），再填E（环境里的其他玩家和规则），最后填P（成功指标）。这个顺序强制你先盘点手段再定目标，避免P膨胀到A够不着。

**成功指标**：四格全部填满，且每个格子有≥3条具体条目（不是"A=写代码"这种笼统项，而是"A=用Cursor写、用Vercel部署、用Sentry监控"这种可操作项）。填完后如果有空格子，设计未完成——不要进入执行阶段。

### 触发器2：反复在同一类问题上卡住

**触发场景**：发现自己在同一模式的问题上反复碰壁——拖延半年还没动、同一类冲突反复发生、每次项目到某个阶段就崩。

**第一个动作**：拿出五层架构图，做"gap诊断"——写下两句话：（a）"我当前主要在哪个层级操作？"（b）"这个问题的复杂度要求哪个层级？"如果（a）低于（b），gap就是根因。例如："我用reflex agent（看到任务→应激处理）面对一个需要goal-based规划（先定义目标状态→反向推导动作序列）的项目管理问题"。

**成功指标**：诊断结果写成"卡在第X层，问题需要第Y层"的格式，且附至少一个具体事例说明为什么第X层不够（例如："我用if-then处理客户投诉——客户说贵就解释、客户不说就不动——但这需要goal-based：先定义'客户关系状态目标'，再反向规划接触节奏"）。不允许输出"我觉得自己不够理性"这种没有层级标签的诊断。

### 触发器3：完成学习活动后的部件更新检查

**触发场景**：读完一本书、上完一门课、完成一个项目、参加完一个会议后——任何"我学完了"的时刻。

**第一个动作**：拿出一张纸，写下这句话并填空："我更新了__（四选一：世界模型/效用函数/动作策略库/评分体系），具体是__从__变成了__，证据是__。"如果不能完成这个填空，你没有学习——你消费了内容。

**成功指标**：能写出至少1条具体修正记录，格式为"在[部件名]中：[旧认知/旧做法]→[新认知/新做法]，证据是[具体经历/数据/反馈]"。"我学到了很多"不算通过——必须指出哪个部件被修改、修改前后的差异、以及触发修改的证据。

### 触发器4：多个目标冲突时的效用函数显式化

**触发场景**：在两个以上相互冲突的目标之间反复纠结——赚钱vs意义、速度vs质量、深度vs广度、家庭vs事业。纠结超过一周没有行动。

**第一个动作**：写效用函数。步骤：（1）列出所有相关变量（≥4个）；（2）给每个变量赋权重，权重之和=1.0；（3）用这个加权公式评估至少3个备选方案。把决策从"跟着感觉反复横跳"变成"给定我的权重，这个方案得分最高"。

**成功指标**：产出至少一个效用公式，权重精确到小数（如0.35不是"重要"），且能向另一个人用≤3分钟解释清楚"为什么健康权重0.4而不是0.3"。允许事后调整权重（这就是learning agent更新效用函数），但不允许"权重差不多就行"——模糊的权重等于没有权重。
