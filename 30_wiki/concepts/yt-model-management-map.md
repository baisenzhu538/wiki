# 一堂管理地图：高潜力管理者修炼全景图

> 来源：一堂《管理地图：高潜力管理者修炼全景图》。[[一堂]] | [[yt-model-progress-map]] | [[yt-system-course-catalog#管理必修]]

## Summary

管理地图是进步大地图中「管理修炼」的完整展开，以「段位」为核心追求。五阶成长路径——管自己→管项目→管团队→管业务→管公司——从个人执行到组织治理逐级爬升。每个阶段有4-5个核心基本功模块，加星号课程为研发中。

## Claims

### 五阶成长路径

| 段位 | 名称 | 核心命题 | 关键模块 |
|------|------|---------|---------|
| **L1** | 管自己 | 突破个人天花板 | 打基础（对接个人修炼地图） |
| **L2** | 管项目 | 拿结果 | 定方案 → 拆计划 → 管执行 → 做复盘 |
| **L3** | 管团队 | 带队伍 | 基本功、懂招聘、会落地、会开会、做沉淀 |
| **L4** | 管业务 | 驱动增长 | 管目标、提业务、做决策、提转化 |
| **L5** | 管公司 | 治理组织 | 做战略、定文化、分股权、管财务 |

### 管项目（L2）：四步闭环

| 步骤 | 课程 | 核心能力 |
|------|------|---------|
| 定方案 | 极简方案设计 | 把模糊需求转化为可执行方案 |
| 拆计划 | 计划拆解基本功 | 把方案拆成可追踪的任务 |
| 管执行 | 管理过程基本功 | 过程中把控进度和质量 |
| 做复盘 | 项目复盘基本功 | 榨干项目认知，加速团队成长 |

### 管团队（L3）：五项基本功

| 模块 | 核心能力 |
|------|---------|
| 基本功 | 带团队练——认知篇 + 关键基本功拆解 + 落地武器库 |
| 懂招聘 | 科学招聘四部曲 + 结构化面试 + 产品经理招聘 |
| 会落地 | 新人 Landing——让新人快速融入和产出 |
| 会开会 | 科学开会认知篇 + 实操篇 |
| 做沉淀 | 知识库——团队知识管理 |

### 管业务（L4）：四大杠杆

| 模块 | 核心能力 | 课程层级 |
|------|---------|---------|
| 管目标 | OKR 认知 + 工具 | 目标管理认知篇 + OKR工具篇 |
| 提业务 | 业务公式拆解 | 导入篇 → 建立公式 → 参数探索 → 逻辑关系 |
| 做决策 | 科学决策 ROI | 审美篇 → 宽度 → 深度 → 高度 |
| 提转化 | 转化率黑客 | 认知篇 → 动力实操 → 阻力实操 → 触点实操 → 组合落地 |

### 管公司（L5）：四大支柱

| 模块 | 核心能力 |
|------|---------|
| 做战略 | 战略会的策划和执行 |
| 定文化 | 公司文化的塑造和落地 |
| 分股权 | 合伙股权认知 + 分配/多元控制/动态调整/签订协议 |
| 管财务 | 财务入门必修 |

### 核心追求

**这张图的核心修炼目标是：段位**——持续提升管理段位，善于用更多资源拿到更大结果。

## Constraints & Boundaries


- claim:boundary-01 [conf=0.80] **五阶路径过于线性**：实际管理场景中，管项目的同时可能已经在管团队，管业务的阶段可能需要回补管团队的基本功——阶跃模型不如螺旋模型准确。
- claim:boundary-02 [conf=0.80] **「段位」评估标准缺失**：什么算 L2 毕业可以进 L3？是管过几个项目还是项目规模？缺乏可操作的评估维度。
- claim:boundary-03 [conf=0.80] **与创业地图存在内容重叠**：管业务（L4）中的「提转化」「做决策」与创业地图增长阶段的「科学决策」「关键转化」重叠——两本姊妹篇的边界需要更清晰。

### 外部攻击：Mintzberg的管理手艺论 + Pfeffer的领导力产业批判

**Henry Mintzberg**（McGill University，"Managers Not MBAs"作者）对管理阶段模型的批评直接适用于管理地图的五阶成长路径。Mintzberg基于29位管理者田野观察提出的核心主张：管理不是一套可以从"管自己"到"管公司"逐级爬升的可编码技能，而是在具体情境中涌现的实践手艺（craft）。五阶模型的"段位"追求暗示"升到L5=更好的管理者"，但Mintzberg会反驳：真正优秀的管理者不是在L5"管公司"的人，而是在任何层级都能根据具体情境做出有效判断的人。把管理拆成阶段课程清单，是把craft变成了checklist。

**Jeffrey Pfeffer**（Stanford GSB，"Leadership BS"作者）对领导力发展产业的经验批评构成第二道攻击。Pfeffer在《Leadership BS》(2015)中系统记录了领导力培训项目的失败模式：它们设计精良、体验良好、参与者满意度高——但几乎不产生实际行为改变。Pfeffer会质问管理地图：学员学完"管团队-L3"的五项基本功后，实际管理行为改变了多少？段位升级的评估标准在哪里？如果没有行为改变的证据，"我从L2升到了L3"不过是一种让人感觉良好的自我叙事。

### 不要用的场景

| 场景 | 为什么失效 | 替代方案 |
|------|-----------|---------|
| 初创公司3-10人团队的管理者 | 五阶模型预设大中型组织的管理分工（管团队/管业务/管公司是分离的），小团队管理者同时在做L2-L5所有事。用五阶模型自评会得出"我什么都在做但哪层都不够"的挫败感——不是能力问题，是框架不适配 | Paul Graham的"Maker's Schedule, Manager's Schedule"——聚焦两种工作模式的节奏切换，而非五阶能力建设 |
| 专业线专家（资深架构师/研究员/设计师）的职业发展 | 管理地图暗示"晋升=从L1管自己爬向L5管公司"，对技术专家是危险误导——最优秀的工程师可能终身在L1-L2（管自己和管项目），专业影响力远超很多L4-L5管理者 | 双轨制职业发展（Dual Career Ladder）——专业线和管理线分开，避免"管理段位=职业段位"的隐含绑定 |
| 扁平化/自组织团队的实践 | 五阶模型预设层级管理范式（管理者vs被管理者），在Holacracy或敏捷自组织团队中，管理职能分布在团队而非集中在"管理者"身上 | Holacracy宪章 + 敏捷教练模式——从"谁管谁"转向"角色和圈层如何自治协作" |

## Action Triggers

| 触发场景 | 第一个动作 | 成功指标 |
|---------|-----------|---------|
| 刚晋升到带团队的角色，不确定该学什么 | 定位你当前在五阶模型的哪一层（L2/L3/L4？），找到对应层的第一个模块（如L3从"带团队基本功"起步），只学这第一个模块，不跳跃 | 在≤10分钟内定位层级，选定1个起始模块并找到学习入口。不允许多选——初带团队最大的错误是同时补所有模块 |
| 管理多年但感觉遇到天花板 | 做"段位gap诊断"——你在哪个L上已经做了很久？下一个L的核心要求是什么？你缺的不是当前L的深度，是上一个L你漏掉的某项基本功 | 写出"我在L[X]→L[X+1]的gap是[具体能力]"的格式，且gap描述可验证（如"L3→L4的gap是'我会带团队但不会拆业务公式'"） |
| 团队出问题，不确定是哪个管理环节的问题 | 用五阶模型的模块做快速扫描——是项目执行（L2）崩了？团队（L3）磨合有问题？业务方向（L4）模糊？先定位层级再深入诊断 | 在≤5分钟内定位到具体层级和模块，标注1条具体现象（如"L3管团队→会开会模块有问题=我们开了很多会但没有决策产出"） |
| 需要给下属做职业发展规划 | 把下属放到五阶模型上定位当前阶段，写出"下一步该突破什么"——不是按层级顺序走，是找他最薄弱的基本功补 | 为下属产出1份≤5行的成长建议，含当前阶段定位+下一个要突破的1个模块+为什么是这个模块 |
