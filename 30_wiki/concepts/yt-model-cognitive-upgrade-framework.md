# 认知升级十步框架：从「读书」到「安装认知补丁」

> 来源：「认知升级系统 v3.2」的底层方法论，经 AI 思维卡（《人工智能：一种现代方法》深度阅读产出）逆向萃取。
> 与 KDO 三步编译法的关系：三步编译法负责**管线**（capture → enrich → produce），十步框架负责**深度阅读场景的质量纵深**——它不是替代，是延伸。

## Summary

认知升级十步框架是一套**把深度阅读转化为认知补丁安装**的结构化方法。十步形成一条完整管道：IDENTITY（这本书到底是什么）→ MODEL（核心补丁长什么样）→ EVIDENCE（这补丁站得住脚吗）→ CONTRAST（什么时候别用）→ ACTION（今晚做什么）→ REFLECTION（什么话留在身上）→ MY TAKE（它对我意味着什么）→ ICAP（我学到了哪一层）→ CTA（三选一最小行动）→ FOLLOWUP（三周后复诊）。核心哲学：**阅读的价值不在"知道了什么"，而在"我的哪个认知部件被更新了"**。

> **Burn line**: 读书不是为了知道更多，而是为了下一次做决定时，脑子里响起的不是旧 Bug 而是新 Patch。

## [Condense]

### 十步流程全景

| 步骤 | 模块 | 核心问题 | 产出物 |
|:--:|------|---------|-------|
| 1 | **IDENTITY** | 这本书到底在说什么？作者是谁？我读之前的旧 Bug 是什么？ | 身份卡（真正在问的问题 + 承诺 + 作者立场 + 旧认知） |
| 2 | **MODEL** | 我装了什么新补丁？核心框架长什么样？ | 3 句精髓 + 5 个工程原语 + 3 条关键判断 + 之前以为 vs 后来发现对照表 |
| 3 | **EVIDENCE** | 这模型站得住脚吗？ | 关键案例审计（含反事实 + 偏差识别）+ 外部攻击收集 |
| 4 | **CONTRAST** | 什么时候用它/什么时候千万别用？ | 与既有方法冲突/互补表 + 适用 vs 失效场景表 |
| 5 | **ACTION** | 我以后到底怎么用？ | 场景触发卡 + 新流程 SOP + 最小可行实验设计 |
| 6 | **REFLECTION** | 什么话留在了身上？ | 3 句原文精炼 + 各自的使用情境 |
| 7 | **MY TAKE** | 这本书对我到底意味着什么？ | 旧认知 → 新认知转化表 + 长期影响的决策场景 |
| 8 | **ICAP** | 我学到了哪一层？下一步该做什么？ | ICAP 四级自评 + 泵升计划 |
| 9 | **CTA** | 今晚就做：三选一 | 3 个 5 分钟内能启动的最小动作 |
| 10 | **FOLLOWUP** | 三周后回来复诊 | 复诊清单（启动了哪些？什么完全没用？什么意外有价值？） |

### 三个差异化设计

**设计 1：Bug → Patch 追踪机制**

每张思维卡强制声明"旧认知（Bug）"和"新认知（Patch）"，外加"让我相信的关键论据"。这是整个框架的核心因果链——不写出旧 Bug，就无法验证是否真的被更新了。

```
旧 Bug："AI 是算法堆叠，跟我做产品没关系"
    ↓ [关键干预：PEAS + 5 层 Agent 架构]
新 Patch："AI 提供了一套普适的智能体设计语言——任何会做决策的系统都可以用 PEAS 描述"
```

**设计 2：EVIDENCE 审计三件套**

不是"找案例证明作者对"，而是：
- **反事实检验**："去掉这个案例，模型还成立吗？"
- **偏差识别**：选择性举证 / 作者偏向 / 过度简化——每条证据标注一个偏差标签
- **外部攻击**："对立阵营会怎么骂这个模型？"——主动收集并呈现

**设计 3：ICAP 自评阶梯**

用 Chi & Wylie 的 ICAP 框架（Passive → Active → Constructive → Interactive）让读者自我定位当前的学习层级，并给出下一步泵升的具体动作。避免"我以为我学到了"的自我欺骗。

### 与 KDO 三步编译法的对标

| 维度 | KDO 三步编译法 | 认知升级十步框架 |
|------|-------------|---------------|
| **定位** | 知识生产管线（从素材到交付） | 深度阅读场景的质量纵深 |
| **粒度** | 原子化卡片 + 图链接 | 单本书的完整加工 |
| **输出** | 多张可独立检索的概念/方法/工具卡 | 1 份自包含的认知补丁安装记录 |
| **复用性** | 卡片可被任意组合引用 | 思维卡是个人产物，不可拆解引用 |
| **适合场景** | 批量知识萃取、课程体系搭建 | 一本想"改变操作系统"的书 |

**互补关系**：三步编译法负责流水线的**宽度**（N 本书 → N 张卡 → 可遍历），十步框架负责单点的**深度**（1 本书 → 1 次行为改变）。两者不是竞争——十步框架深度加工后的素材，可以拆解为多张 KDO 卡片入库。

## [Critique]

### 内部局限性

**1. 制作成本极高**

一张完整思维卡的产出需要数小时（读原书 + 提取框架 + 证据审计 + 边界地图 + 实验设计）。对信息类/消遣类阅读严重过度。只适合"这本书想改变我的操作系统"级别的投入——每年可能只有 3-5 本。

**2. 依赖较强的领域抽象能力**

MODEL 模块要求把一本书压缩为"3 句精髓 + 5 个工程原语"——这需要能区分骨架和噪音。如果读者本身缺乏该领域的认知框架，可能提取出错误的骨架。

**3. ICAP 自评的客观性存疑**

自我评估学习层级（"我到了 Constructive 层"）天然存在高估风险。没有外部校准机制（如第三方测试或专家反馈）的 ICAP 自评，准确度上限受限于自我认知水平。

### 外部攻击

**攻击 1：认知心理学家对"补丁安装"隐喻的批评**

Daniel Willingham（《为什么学生不喜欢上学》）的核心论点：认知不是软件升级——你不能像打补丁一样"安装"一个思维模型然后期望行为自动改变。真正的认知变化需要大量情境化练习和反馈循环，不是一次深度阅读能完成的。十步框架的 CTA 和 FOLLOWUP 试图解决这个问题，但 CTA 的 5 分钟最小动作可能给人"我已经在改了"的虚假安全感。

**攻击 2：纳西姆·塔勒布对"框架化一切"的警告**

塔勒布在《反脆弱》中反复警告：过度理论化会让人对现实的复杂性视而不见。十步框架把阅读变成高度结构化的工程流程——这可能系统性地过滤掉那些"无法被归入框架但很重要"的洞见。尤其是 CONTRAST 的"不要用的场景"表，如果写表的人本身就在该框架的思维范式内，他写出的"边界"不过是框架内部的盲区。

### 不要用的场景

| 场景 | 为什么失效 | 替代方案 |
|------|-----------|---------|
| 阅读新闻/博客/碎片信息 | 十步框架的投入产出比需要"这本书值得花 3-8 小时加工"。碎片化内容用十步框架就像用手术刀削苹果——工具过重 | 清单体笔记（[[yt-personal-checklist-notes]]）或渐进总结——30 秒抓关键点即可 |
| 小说/文学/艺术类阅读 | 这些文本的核心价值不在"可提取的模型/框架"，而在体验、情感和美学共鸣。强行 EVIDENCE 审计会杀死阅读的快感 | 费曼笔记（用一句话讲给别人听）或完全不做笔记——有些书就是为了沉浸，不是为了升级 |
| 你已经深度掌握该领域 | 如果你已经是该领域的 Constructive/Interactive 层——你不需要再用十步框架读一本入门书。你会花 8 小时写你已经知道的东西 | 直接跳到 CONTRAST（快速对比新旧框架的差异点）或写一篇"这本书对我的 3 个新信息"即可 |

### 知识体系定位

十步框架在 KDO 知识生产管线中的位置：

```
capture → ingest → enrich ─────────→ produce → validate → ship
                         ↑                        ↑
                    十步框架介入点          三步编译法 assemble
                    （深度阅读场景）
```

三步编译法负责管线（宽度），十步框架负责深度阅读的质量纵深（深度）。两者配合：十步框架加工后的素材 → 拆解为多张 KDO 卡片 → 入库图链接网络。

## Action Triggers

| 触发场景 | 第一个动作 | 成功指标 |
|---------|-----------|---------|
| 读完一本"感觉很重要但不知道从哪用起"的书 | 打开 Obsidian 新建一页，先填 IDENTITY 的 5 张身份卡（真正在问的问题 / 承诺 / 作者 / 版本 / 旧 Bug）。**不要跳到 MODEL——IDENTITY 写不满别往下走** | ≤30 分钟完成 IDENTITY 模块；至少写出 2 条"旧 Bug"且每条对应一个过去的真实决策失误 |
| 有人问你"最近读了什么好书？"而你说不出具体学到了什么 | 拿出一张纸，画 Bug → Patch → 让我相信的关键论据 三列。**逼自己填——填不满就承认这本书只是翻了翻** | 3 分钟内完成三列；Patch 那列至少有 1 条能直接指导明天的某个具体动作 |
| 你想判断一本还没读的书值不值得深度加工（避免花 8 小时在烂书上） | 用 10 分钟翻目录 + 前言 + 随机 3 页 → 试填 IDENTITY 的"它真正在问的问题"和"它向我承诺什么"。**如果这两格填不出——这本书不值得十步加工** | 10 分钟内做出 Go/No-Go 决策；Go 的标准是两格都填得具体且让你兴奋 |
