# 产品内核十大典型指标：从获客到推荐的度量框架

> 来源：一堂知识地图《产品内核·十大典型指标》。[[yt-entrepreneur-product-core]] | [[yt-entrepreneur-unit-model]] | [[一堂]]

## Summary

十大典型指标是一堂产品内核体系的核心度量框架——按用户全生命周期分四个环节：获客→服务→复购→推荐，每个环节配置 2-3 个关键指标。这套指标不是"越多越好"的数据看板，而是"做产品只盯这十个就够了"的极简度量哲学。核心逻辑：产品内核的质量由少数关键转化率决定，找到它们、盯住它们、优化它们。

## Claims

### 四大环节 × 十大指标

| 环节 | 指标 | 一句话说明 |
|------|------|-----------|
| **获客** | 销转率 | 潜在客户转化为实际购买客户的比例 |
| **获客** | 动销率 | 有销售的商品品种数 / 所有商品总品种数 |
| **获客** | 捕获率 | 进店消费人数 / 总经过人流量 |
| **服务** | 留存率 | 使用产品 N 天/周/月后还在持续使用的比例 |
| **服务** | 完课率 | 用户是否能完成履约和最后学习（教育类） |
| **服务** | 退款率 | 服务履约中用户退款的比例（反向指标） |
| **服务** | 满意率 | 用户接受完服务后的满意度打分 |
| **复购** | 复购率 | 消费购买后会继续购买的比例 |
| **复购** | 续费率 | 续费用户数 / 现有用户数 |
| **推荐** | 推荐率 | 用户是否愿意主动推荐的比率（NPS 核心） |

### 使用原则

- **不做大而全的看板**：十个指标足矣——它们覆盖了用户全生命周期
- **找到你的"关键转化率"**：不是十个全盯，而是找到你的业务中最关键的 1-3 个
- **获客→服务→复购→推荐**是一个漏斗，但更是四个可独立优化的模块
- 指标之间互相联动：留存率差 → 推荐率不可能高；退款率高 → 复购率不可能高

### 与产品内核的关系

产品内核 = 决定产品质量的 1-3 个关键指标。十大指标是"候选池"，产品内核是你从这十个（或类似指标中）选出的 1-3 个核心：

> 目标用户 × 核心指标 × 内核假设 = 产品内核画布

## Visual Analysis

**视觉结构**：四个环节从左到右横向排布——获客→服务→复购→推荐——形成完整的用户生命周期流。这是一个漏斗隐喻的倒置：不是"越往下越少"的悲观漏斗，而是"每个环节都可以独立做大"的乐观流水线。环节之间的连接线暗示指标之间的传递关系（留存率差→推荐率不可能高）。

**分组逻辑**：获客 3 指标（销转率/动销率/捕获率）覆盖"用户从哪来"，服务 4 指标（留存率/完课率/退款率/满意率）覆盖"用户是否留下"，复购 2 指标（复购率/续费率）覆盖"用户是否回来"，推荐 1 指标（推荐率）覆盖"用户是否带人来"。指标数量从获客→推荐递减（3→4→2→1），暗示越往后环节指标越收敛到单一核心指标。

**刻意强调**：指标名称全是"X率"格式——统一命名暗示这是一套可比较的度量体系，不是散装 KPI。每个指标配"一句话说明"而非公式，暗示"先理解含义，再讨论计算方法"——这是面向创业者而非数据分析师的表达策略。

## Constraints & Boundaries


- claim:boundary-01 [conf=0.80] **十个指标偏教育/ SaaS 视角**：完课率、续费率等指标在教育/SaaS 行业适用，但对餐饮（翻台率）、电商（GMV/客单价）、内容（完播率/互动率）等行业缺位。框架通用性需要行业适配层。

- claim:boundary-02 [conf=0.80] **只列指标不教归因**：知道"留存率低"不难，难的是知道"是因为 onboarding 差还是核心价值没交付"。十大指标是指示灯，不替代归因分析。
