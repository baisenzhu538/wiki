# Kimi 深度调研集群方法论 (Deep-Research-Swarm)

## Summary

> **核心理念**: 多智能体认知三角测量（Epistemic Triangulation）——在不同研究维度上发散、检测重叠与矛盾、深度验证、再收敛为经过验证的综合结论。

集群并行服务于认知鲁棒性，而非仅仅追求速度。

所有调研任务首先经过**意图与输入路由器**，根据任务特征自动分类为四条路线： | 路线 | 适用场景 | 特征 | |------|---------|------| | **Route A — 广域搜索** | 宽泛/探索性主题，维度不明确 | 先广域探索，再深度挖掘；两阶段并行集群 | | **Route B — 聚焦搜索** | 具体问题，维度清晰 | 标准流程：概览扫描 → 分解 → 并行深度挖掘 | | **Route C — 纯文件研究** | 用户明确要求仅基于上传文件 | **零外部搜索**；多文件深度分析 → 交叉验证 → 写作 | | **Route D — 文件增强研究** | 文件为主要参考，允许外部补充 | 先分析文件，再针对空白进行外部搜索，融合输出 | **分类信号**： - 文件 + "仅基于文件" / "不上网搜索" → **Route C** - 文件 + "参考这些文件" / "结合文件" / 无明确限制 → **Route D** - 无文件 + 宽泛/探索性主题 → **Route A** - 无文件 + 具体/有界问题 → **Route B** ``` 用户查询   │   ▼ Phase 0: 意图与输入路由   │   ├─ Route A: 广域搜索   │   → Phase 1 (快速概览) → Phase 1W (多Agent广域探索)   │     → Phase 2 (维度分解) → Phase 3 (并行深度挖掘)   │       → Phase 4 (交叉验证) → Phase 5 (如有冲突)   │         → Phase 6 (洞察提取) → Phase 7 (移交写作)   │   ├─ Route B: 聚焦搜索   │   → Phase 1 (概览扫描) → Phase 2 (维度分解)   │     → Phase 3 (并行深度挖掘) → Phase 4 (交叉验证)   │       → Phase 5 (如有冲突) → Phase 6 (洞察提取) → Phase 7   │   ├─ Route C: 纯文件研究   │   → Phase F (文件 intake 与深度分析)   │     → Phase 2 (从文件主题分解维度)   │       → Phase 3-F (多Agent文件深度挖掘，无外部搜索)   │         → Phase 4 (跨文件交叉验证) → Phase 6 → Phase 7   │   └─ Route D: 文件增强研究       → Phase F (文件 intake 与深度分析)         → Phase 1 (针对性概览，填补文件空白)           → Phase 2 (融合文件主题 + 外部概览分解维度)             → Phase 3 (并行深度挖掘，文件上下文 + 搜索)               → Phase 4 → Phase 5 → Phase 6 → Phase 7 ``` 在任何分析或叙述生成之前，系统必须： - **假设内部知识可能过时或不完整** —— 始终先获取当前日期/时间 - **时间敏感性约束** —— 当用户查询含时间要求（如"2026 Q1"、"最近6个月"），将指定时间范围视为硬约束，搜索查询必须 targeting 该窗口 - **外部证据优先** —— 先进行外部广泛搜索建立证据图景（Route C 除外） - **禁止先搜索后叙事** —— 在搜索/文件分析输出前不得生成任何事实性声明 - **搜索语言规则** —— 所有搜索查询必须与用户消息语言一致（中文查询用中文搜索，英文查询用英文搜索） - **所有输出必须使用内联引用 `[^number^]` 标注原始来源** **目标**: 从所有上传文件中提取结构化知识，构建证据地图，识别主题、声明、矛盾与空白。
