# 双三角竞争力模型 (Yitang Dual Triangle Model)

> 来源：一堂《AI 时代的竞争力·双三角模型》（2025.9.5 北京一堂实验室）。[[一堂]] | [[一堂方法论体系总图]]

## Summary

双三角模型回答两个问题：（1）同样会用 AI，为什么不同人业务结果相差巨大？（2）同样都是业务高手，为什么不同人 AI 结果相差巨大？答案在于人类三角（审美→体系→基本功）与 AI 三角（数据→场景→创造力）的乘积效应——人变强了调教更强 AI，有了强 AI 辅助人变得更强。

## Claims

### 人类三角 × AI 三角

```
人类三角                    AI 三角
┌─────────────┐           ┌─────────────┐
│   创造力    │←─乘积效应─→│   创造力    │
│   (场景)    │           │   (场景)    │
├─────────────┤           ├─────────────┤
│   审美      │           │   数据      │
│ ·最佳实践   │           │ ·宽度：多模态/工具库(8)
│ ·评价标准   │           │ ·结构化数据
│ ·假设验证模型│          │ ·形成资产   │
├─────────────┤           ├─────────────┤
│   体系      │           │   场景      │
│ ·科学体系思维│          │ ·深度：模型微调训练/工程化
│ ·稳定知识结构│          │ ·非结构化数据│
│ ·基本功     │           │ ·业务拆解    │
│ ·理解底层规律│           │ ·ROI 预判   │
│ ·善于解放思想│           │ ·AI 打磨    │
│ ·组织部署    │           │ ·提出创造假设│
└─────────────┘           └─────────────┘
```

### 双螺旋增强回路

1. **人变强了 → 调教更强 AI**：审美提升 → 更好的评价标准 → 更精准的 AI 指令
2. **有了强 AI → 辅助人变得更强**：AI 输出高质量 → 拉升人的见识上限 → 循环迭代

### 基础层

- 人类三角基础：LLM 提示词/智能体
- AI 三角基础：专业见识、科学体系思维

## Constraints & Boundaries


- claim:boundary-01 [conf=0.80] **乘积效应的量化证据缺失**：声称「乘积效应」但未提供可测量的交互增益数据——是乘法还是加法，需要实证。
- claim:boundary-02 [conf=0.80] **「基本功」「审美」定义边界模糊**：审美体系与基本功在实操中的区分不够清晰，尤其在「最佳实践 → 评价标准」这条线上重叠严重。
- claim:boundary-03 [conf=0.80] **模型偏向 AI Native 场景**：对于传统行业（制造业、农业等），AI 三角中的「场景」和「数据」要素的表达力可能不足。

### 外部攻击：Henry Mintzberg的实践手艺论 + Nassim Taleb的幸存者偏差

**Henry Mintzberg**（管理学家，McGill University，"Managers Not MBAs"作者）对管理能力框架的经典批评直接适用于双三角模型：将竞争力拆解为"审美×体系×基本功"等结构化维度，犯了管理教育的原型错误——把实践智慧（craft）分解为可教授的离散技能，制造"知道框架=具备能力"的认知幻觉。Mintzberg在《Managing》(2009)中基于29位管理者的田野观察提出，真正有效的管理能力是在具体情境中涌现的实践手艺，无法被前置编码为六维矩阵。双三角框架让学员感知到"我在系统地提升竞争力"，但框架的对称美学可能掩盖实际竞争力的高度情境依赖性和非结构化特征。

**Nassim Taleb**（《Antifragile》作者）的幸存者偏差批评：从成功者身上逆向提取"人类三角+AI三角"要素，犯了"在赢家身上找共同特征"的归纳错误。Taleb会问：有多少同时具备审美/体系/AI能力的人失败了？没有失败者控制组的竞争力归因，可能是模式识别幻觉——你总能在一群成功者身上找到共同特征，但这不等于因果关系。

### 不要用的场景

| 场景 | 为什么失效 | 替代方案 |
|------|-----------|---------|
| 纯执行层个体贡献者（初级工程师/设计师/分析师）的职业发展 | 此类角色的竞争力瓶颈在单一技能的刻意练习深度，而非多维度三角平衡。框架的"乘积效应"提示会让他们把注意力分散到尚未掌握的维度上，导致每维浅尝辄止 | Ericsson刻意练习框架——聚焦单一技能的突破性提升，等该维度达到可迁移水平后再引入多维整合 |
| 成熟组织的人力盘点或晋升决策 | 双三角是学习框架而非测量工具，六维度缺乏可操作的评分量表。"审美3分、基本功4分"式的评分是伪精确，在人事决策中的解释力远低于可验证的行为事例 | Laszlo Bock（Google前HR VP）的"Work Rules!"绩效评估体系——基于可验证的行为事例（behavioral examples）而非维度评分 |
| 硬科技/深度科研领域（制药、半导体、基础物理）的战略分析 | 这些领域的竞争力由专利壁垒、实验设备和专业知识深度决定，而非"人类三角×AI三角"的乘积逻辑。用双三角分析制药公司竞争力会得出"他们需要提升场景创造力"的无效建议 | Teece动态能力框架（Dynamic Capabilities）——从资源基础观出发分析组织的感知（sensing）、抓取（seizing）、转型（transforming）能力 |

## Action Triggers

| 触发场景 | 第一个动作 | 成功指标 |
|---------|-----------|---------|
| 准备年度个人发展计划或学习预算分配 | 画双三角六维自评雷达图，每个维度打分（1-5），必须提供过去3个月的具体行为事例作证据，标出最低的两个维度 | 六维全部完成评分且每个分数后有≥1条过去90天的行为事例。出现"不知道怎么打分"的维度则标记为"盲区"优先探索 |
| 团队建设/招人时纠结"该补什么能力" | 把团队每个人的双三角画出来，找"集体盲区"——哪个维度全员缺失或都弱。先补盲区，再补深度 | 产出团队双三角热力图，盲区维度被标注且下一轮招聘JD中明确写入该维度要求 |
| AI工具用了一段时间没感觉产出提升 | 用双三角诊断AI三角——是数据积累不够？没找到真正的业务场景？还是创造力没被AI放大？定位薄弱维度 | 定位到AI三角中≥1个具体薄弱维度，并在两周内对该维度做≥1次针对性实验（如场景薄弱→列10个可能场景并测试3个） |
| 业务遇到瓶颈，尝试多个方案效果不佳 | 回到人类三角自查——是审美缺失（不知道什么是好的）？体系缺失（知道好的但没有系统方法做到）？还是基本功缺失（知道方法但执行不到位）？ | 定位瓶颈在人类三角的具体角，并有≥1个具体现象支撑（如"审美缺失=我们看了竞品但说不出对方好在哪里，评价标准是'感觉'"） |
