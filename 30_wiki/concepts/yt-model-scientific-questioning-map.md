# 科学提问刻意练习地图

> 来源：一堂《科学提问·刻意练习成长地图》（内部使用）。[[一堂]] | [[yt-personal-deliberate-practice]] | [[yt-personal-deep-review]]

## Summary

一堂将提问能力按四个领域（业务分析→辅导团队→咨询教练→萃取经验）和三级难度分层，提供了每层的目标、场景、套路和工具箱。核心主张：提问不是天赋，是可以通过结构化模型刻意练习获得的核心能力。

## Claims

### 四领域 × 三级难度矩阵

| 领域 | 面向 | Level 1 | Level 2 | Level 3 |
|------|------|---------|---------|---------|
| **业务分析** (Analysis) | 面向业务 | 理解问题 → 共同评估 → 引导建模 | 对齐目标 → 强迫思考 → TopN 优先级 | 方案讨论 → 分析问题 → 交叉验证 |
| **辅导团队** (Coaching) | 面向团队 | 下属沟通 → 达成共识 → 代入场景 | 能力辅导 → 建设性提问 → 绩效沟通 | 面试评估 → 辩证讨论 |
| **咨询教练** (Teaching) | 面向客户 | 付费咨询 → 内核边界 → 关键假设 | 创业教练 → 苏格拉底式提问 | 私董会 → 探讨解法 |
| **萃取经验** (Socrates) | 面向专家 | 帮助客户提升找答案能力 | 公司萃取知识 | 个人做课 |

### 通用工具箱

- **业务分析**：STAR 模型、加减法、MECE 拆解
- **辅导团队**：GROW 模型、追问榨干经验、六个 Why
- **咨询教练**：关键假设、代入问题场景、苏格拉底式提问

### 提问的核心维度

| 维度 | 说明 |
|------|------|
| 提问目标 | 找出问题答案 / 提升员工找答案能力 |
| 提问对象 | 面向业务 / 面向团队 / 面向客户 / 面向专家 |
| 提问难度 | 青铜→白银→黄金→钻石→王者 |
| 使用场景 | 能力辅导、面试评估、绩效沟通、付费咨询、创业教练、私董会等 |

## Constraints & Boundaries


- claim:boundary-01 [conf=0.80] **四领域之间的交叉训练路径缺失**：业务分析能力与辅导能力如何在同一个练习循环中交替提升，未说明。
- claim:boundary-02 [conf=0.80] **难度分级的操作性不足**：青铜/白银/黄金/钻石/王者的区分标准过于模糊，需要可量化的晋级条件。
- claim:boundary-03 [conf=0.80] **工具箱的领域特异性**：STAR 和 GROW 跨领域使用是否有效，还是它们在各自领域内有不可替代的适配性。

## Framework Gallery

### 关联框架卡

### 外部攻击：Paulo Freire的提问式教育 + 苏格拉底本人的反方法化

**Paulo Freire**（"Pedagogy of the Oppressed"作者，20世纪最有影响力的教育哲学家之一）对"提问作为教学方法"的论述与科学提问地图的结构化方法形成根本张力。Freire在《被压迫者教育学》(1968)中区分了"银行式教育"（教师存入、学生取出）和"提问式教育"（师生在对话中共同创造知识）。Freire的核心洞见：提问的力量来自"对话者的真实好奇和共同探索"，而非"提问者掌握了一套提问技术"。科学提问地图把提问拆解为"四领域×三级难度矩阵+6个工具箱"，但Freire会说：当你把提问变成一套可训练的技术时，你失去了提问最本质的东西——提问者对被问者作为平等主体的尊重。真正的提问不是"我用GROW模型辅导你"，而是"我真的想知道你怎么看"。

**苏格拉底本人**的反方法化是一个有趣的元批判。苏格拉底的提问法（elenchus）之所以强大，恰恰因为它不是一套可以写成"步骤1→步骤2→步骤3"的方法——每一次苏格拉底式提问都是针对对话者具体回答的即兴追问。苏格拉底在柏拉图的对话录中反复强调自己"一无所知"（aporia），而方法化苏格拉底式提问（把它的"核心套路"和"工具箱"写成卡片）恰恰背叛了苏格拉底的核心精神——当你觉得你掌握了苏格拉底式提问的方法时，你就进入了苏格拉底警告的"以为自己知道"的状态。

### 不要用的场景

| 场景 | 为什么失效 | 替代方案 |
|------|-----------|---------|
| 创意发散/头脑风暴阶段 | 科学提问地图的结构化追问（STAR/GROW/苏格拉底）是收敛工具，不是发散工具。在需要"疯狂的想法""天马行空"的创意阶段，结构化提问会把思维过早钉死在框架上 | IDEO的brainstorming规则——延迟判断、追求数量、鼓励wild ideas、在他人想法上搭建——不追问"为什么"只追问"还有什么" |
| 与孩子的对话/家庭教育场景 | 科学提问地图的"辅导团队"层工具箱（GROW/追问榨干经验/六个Why）预设被问者是成熟理性主体。对孩子来说，连问三个"为什么"的感受不是"他在辅导我"而是"他在审问我" | 开放式游戏提问——"如果X会怎样？""你觉得故事接下来会发生什么？""你最喜欢哪个部分？为什么？"——不追求结构，追求想象 |
| 被问者已经有明确答案、只需要确认/授权时 | 地图预设"提问的目的是帮助对方找到答案"。但如果对方已经有答案了，再用GROW或苏格拉底式提问走一遍流程是无效仪式——对方会觉得"我都知道答案了你在绕什么" | 直接确认——"听起来你已经有了答案，说说你的想法？""你需要我做什么来支持你？"——尊重效率而非执行问询流程 |

## Action Triggers

| 触发场景 | 第一个动作 | 成功指标 |
|---------|-----------|---------|
| 辅导团队中的新人/下属时，不知道从哪里开始问 | 先定位领域——这是业务分析（L1）、辅导团队（L2）、咨询教练（L3）还是萃取经验（L4）？选对领域后再选工具箱中最简单的1个工具——如L2用GROW | 在≤2分钟内定位领域并选定1个工具。如果定位不了=你还没理解这个辅导场景的本质 |
| 做需求访谈/用户研究 | 用业务分析（L1）的MECE拆解——把你要了解的领域拆成互不重叠的子问题，每个子问题设计1-2个开放式问题。不设计"你觉得这个功能好不好"式问题 | 访谈前设计≥10个问题，全部是开放式（不能用"是/否"回答），且≥5个问题是追问型（"你上次做X时具体做了什么？然后呢？"） |
| 被问到复杂问题，第一反应是给答案而非提问 | 练习"先问一个问题再给答案"——听到问题后，默数3秒，说"在我回答之前，我想先确认一个事情：你说的X是指...？"（用提问确认理解） | 一天中≥3次在给答案之前先问了一个澄清问题。至少1次通过澄清问题发现"对方问的和他真正需要的不一样" |
| 个人技能复盘——"我的提问能力如何？" | 做提问频率审计——过去7天的会议/对话中，你说话时间的比例是多少？如果>60%，你不是在提问是在讲。你的问题中"开放式"和"封闭式"的比例是多少？ | 产出自评数据：说话占比+开放vs封闭问题比例。说话>50%→下个7天目标说话<40%。封闭式>50%→每天练习把≥3个封闭式问题改为开放式 |
