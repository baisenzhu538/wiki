# 泛产品设计工具篇：出牌教学法与30张卡牌

> 来源：一堂泛产品设计框架篇口述（Truman）。[[yt-model-pan-product-36-strategies]] | [[yt-personal-pan-product-concepts]] | [[一堂]]

## Summary

工具篇是泛产品设计课题的"出牌教学法"首创课——首次将 Truman 十年积累的30+张产品工具以扑克卡牌形式组织教学。核心洞见：做产品做不好，是因为"手里的牌太少"——面对大大小小的产品设计任务，需要多张可组合的"牌"（工具/原则/方法论）。三组牌（黄/绿/蓝 = 需求/审美/落地）从 Truman 身上被"凶残地榨取"出来，小到写一封信，大到架构创新业务，都是这30张牌反复出。

## [Condense]

### 出牌教学法

核心理念：产品设计是一种"出牌能力"——根据任务场景选择合适的卡牌组合。

| 教学突破 | 详情 |
|---------|------|
| **问题诊断** | 做不好产品是因为工具太少——手里的牌太少 |
| **解决方案** | 把隐性工具显性化——从Truman身上榨出30张牌 |
| **组织形式** | 三组色系（黄=需求/绿=审美/蓝=落地），每组多张 |
| **使用方式** | 小任务出2-3张，大项目出十几张 |

### 30张牌的来源

- 三分之一到一半：在其他一堂课里讲过（如五步法、动力阻力触点）
- 一半以上：Truman 个人工具，此前从未在课上系统出现
- 来源方式：团队"对 Truman 非常凶残的一次一次榨取"

### 三组牌的逻辑

```
黄色系（需求）    →  回答"为谁做、为什么做"
绿色系（审美）    →  回答"做成什么样算好"  
蓝色系（落地）    →  回答"怎么磨出来"

任务从小到大：先出需求牌理解用户 → 出审美牌定标准 → 出落地牌磨出来
```

### 关键教学决策

- **为什么用卡牌？** 抽象工具太干，卡牌形式降低认知负担，容易记住和组合
- **为什么是30张？** 不是理论推导，而是实操验证——这些是 Truman 实际反复使用的工具
- **一张牌 vs 一节课**：每张牌都值得讲一节课甚至多节课（如五步法本身就是一个完整课题），框架篇只能做导览

## [Critique]

1. **30张牌的认知负荷巨大**：一节课塞入30张牌，虽然用了出牌教学法降低门槛，但学员可能"看完热闹记不住牌"。需要后续实操篇逐张展开才能真正内化。

2. **"被榨取"的隐性知识外化过程未透明化**：Truman 说自己被团队"凶残榨取"，但这个外化过程本身就是知识萃取的最佳案例——如果能展示"某张牌是怎么被榨出来的"，教学效果可能更好。
