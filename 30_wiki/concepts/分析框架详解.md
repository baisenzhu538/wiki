# 分析框架详解

## Summary

用于 Step 7：竞争格局与关键发现分析。

根据调研类型选择对应框架。

``` 企业画像 ├── 基本信息（成立时间、注册资本、实控人、融资轮次） ├── 业务版图（产品线、业务线、客户群） ├── 财务健康度（营收、利润、现金流、负债率） ├── 团队实力（创始人背景、核心团队、人才策略） ├── 技术壁垒（专利、技术栈、研发投入） ├── 市场表现（市占率、增速、用户数据） ├── 合规风险（诉讼、处罚、合规问题） └── 战略信号（招聘方向、新产品、合作/并购动态） ``` | 维度 | 竞品A | 竞品B | 竞品C | 我方 | |------|-------|-------|-------|------| | 产品定位 | | | | | | 核心功能差异 | | | | | | 定价策略 | | | | | | 目标用户 | | | | | | 用户规模/增速 | | | | | | 获客渠道 | | | | | | 口碑/NPS | | | | | | 技术壁垒 | | | | | | 团队规模 | | | | | | 融资状况 | | | | | | 核心优势 | | | | | | 主要短板 | | | | | ``` 行业全景 ├── 市场规模与增速（TAM/SAM/SOM） ├── 竞争格局（集中度、头部玩家、新入场者） ├── 产业链结构（上游、中游、下游、利润分布） ├── 商业模式（主流模式、盈利方式、单位经济模型） ├── 用户画像（规模、特征、需求、付费意愿） ├── 驱动因素（技术/政策/消费/资本） ├── 风险因素（政策风险、竞争风险、技术风险） └── 趋势判断（短期/中期/长期 + 置信度） ``` ``` 项目评估 ├── 商业模式可行性（单位经济模型、回本周期） ├── 品牌/团队可信度（背景、历史、口碑） ├── 市场空间（是否还有增量机会） ├── 合规风险（法律诉讼、加盟纠纷、监管风险） ├── 财务健康度（如有公开数据） ├── 现有参与者满意度（加盟商/已投项目表现） ├── 竞争壁垒（护城河分析） └── 退出机制（如不成功，损失范围和退出路径） ``` | 因素 | 当前状态 | 趋势 | 影响强度 | 对调研对象的影响 | |------|---------|------|---------|----------------| | **P** 政策/政治 | | ↑/↓/→ | 高/中/低 | | | **E** 经济 | | ↑/↓/→ | 高/中/低 | | | **S** 社会/文化 | | ↑/↓/→ | 高/中/低 | | | **T** 技术 | | ↑/↓/→ | 高/中/低 | | | **E** 环境/ESG | | ↑/↓/→ | 高/中/低 | | | **L** 法律/监管 | | ↑/↓/→ | 高/中/低 | | > 关键提醒：PESTEL 因素之间存在**级联效应**（如政策变化 → 技术路线调整 → 市场格局变化），因素间的互动往往比单个因素更重要。
