# PEAS 智能体分析：用 AI 工程师的眼睛诊断任何决策系统

> 来源：Russell & Norvig《Artificial Intelligence: A Modern Approach》第 4 版第 2 章。本卡将 PEAS 四元组 + 5 层 Agent 架构打包为一张可独立使用的自我诊断工具卡。

## Summary

PEAS 是描述**任何决策系统**的通用工程语言——不只是 AI，也适用于你自己、你的团队、你的项目。四个槽位（Performance 性能度量 / Environment 环境 / Actuators 执行器 / Sensors 传感器）把模糊目标拆成可诊断的工程问题。5 层 Agent 架构（Simple Reflex → Learning Agent）告诉你"卡住了该升级哪个部件"。核心洞见：**拖延不是性格问题，是 reflex agent 在打 multi-agent 动态战——架构错配**。

## [Condense]

### PEAS 四元组速查

| 字母 | 工程含义 | 自我诊断问题 |
|------|---------|------------|
| **P**erformance | 性能度量——成功长什么样？ | 谁打分？几个指标？打完之后会怎样？ |
| **E**nvironment | 环境——在哪儿玩？ | 谁是其他 agent？是合作还是博弈？环境会因我的动作改变吗？ |
| **A**ctuators | 执行器——能做什么？ | 我的动作集是什么？哪些直接可用？哪些要审批？ |
| **S**ensors | 传感器——能感知什么？ | 反馈从哪来？实时还是延迟？盲区在哪？ |

**90% 的 OKR 失败是因为只填了 P（目标数字），漏了 E/A/S**。定了"年营收 X 亿"但没定义市场环境、没列出可用手段、没设计反馈机制——不是没努力，是被设计成失败。

### 任务环境 7 维分类

| 维度 | 低难度端 | 高难度端 |
|------|---------|---------|
| 可观察性 | 完全可观察 | **部分可观察** ← 真实世界 |
| 确定性 | 确定性 | **随机** ← 真实世界 |
| 状态空间 | 离散 | **连续** ← 真实世界 |
| 动态性 | 静态 | **动态** ← 真实世界 |
| Agent 数量 | 单 agent | **多 agent** ← 真实世界 |
| 已知性 | 已知 | **未知** ← 真实世界 |
| 时间结构 | 回合制 | **序列决策** ← 真实世界 |

学校是左列（温室），职场是右列（战场）。**从学校到职场让人怀疑人生，是因为你被训练成下棋手，结果被扔进了战场**。

### 5 层 Agent 架构：自我诊断地图

```
Level 1 — Simple Reflex        "看到刺激就反应"           没有记忆、没有规划
Level 2 — Model-based Reflex   "脑内有世界模型再反应"      知道传感器看不到的地方可能发生什么
Level 3 — Goal-based           "为达成目标做规划"          能从当前状态反向规划到目标状态
Level 4 — Utility-based        "在多个目标间权衡取舍"       有明确的效用函数做 trade-off
Level 5 — Learning Agent       "持续更新自己的某个部件"     能指出"我这周更新了哪个模块"
```

**关键跃迁不是"更努力"，是换架构**。一个 reflex 思考的人，再怎么学方法论也变不成战略思考者——除非他换架构。

### 理性决策 = 在已知信息下最大化预期效用

- 不要求**结果**最优，只要求**决策**最优——区分决策质量和结果质量
- 信息有限是默认设置——你不是先知
- 效用要先定义清楚——这就是 P
- 抱怨环境（E）是徒劳——E 是给定的，你能动的只有 S（拿更好的信息）和 A（练更精的动作）

### 场景速查表

| 触发场景 | 诊断框架 | 典型发现 |
|---------|---------|---------|
| 接到模糊新项目 | PEAS 四元组 | P 空槽 → 先别做，找能定义 P 的人 |
| 某件事反复做不好 | Agent 架构层级 | reflex 在打 multi-agent 动态战 → 需要升到 model-based 或 goal-based |
| 重大不可逆决策 | 效用函数展开 | 隐藏变量（部分可观察）→ 先做信息收集动作再决 |
| 感觉"做了很多但没进步" | Agent 类型周复盘 | reflex 占比 > 50% → 下周强制安排 utility 决策时间块 |

## [Critique]

### 内部局限性

**1. PEAS 假设 P 可定义**

PEAS 默认性能度量可以被明确写出。但人生最难的问题恰恰是 P 没人能给——养育、亲密关系、艺术、伦理。在这些领域强行 PEAS 化会把丰富目标退化为可测但失真的代理指标（古德哈特定律：当一个指标成为目标，它就不再是好指标）。

**2. 5 层架构不完全适用于 LLM 时代**

现实中的大语言模型是混合体——同时包含 reflex（模式匹配）、model-based（世界知识）和 goal-based（指令遵循）元素，不完全落在单一层级。五层架构作为思维工具仍然有效，但不应用于评判或预测 AI 系统的具体能力边界。

### 外部攻击

**攻击 1：价值多元主义者对效用可累加性的挑战**

以赛亚·伯林（Isaiah Berlin）和当代价值多元论者认为：不是所有价值都可以放在同一把尺子上比较。"家庭时间"和"职业成就"可能根本不可通约（incommensurable）——强行把它们转化为"效用"的比较，不是在"做更好的决策"，而是在"逃避做选择的不适感"。PEAS 的理性框架把不可通约的选择伪装成了可计算的问题。

**攻击 2：加里·克莱因的自然决策理论**

克莱因（Gary Klein）研究消防员、军人和急诊医生的真实决策过程后发现：专家在高压下根本不做什么"效用最大化"——他们靠**识别启动决策**（Recognition-Primed Decision）：看到情境 → 匹配脑内模式 → 模拟一个方案 → 如果可行就执行。他们不会列出选项比较效用。PEAS + 理性决策框架描述的是"学生做题"式的决策，不是真实世界专家的决策。

### 不要用的场景

| 场景 | 为什么失效 | 替代方案 |
|------|-----------|---------|
| P 本身在争议中的决策——养孩子、做艺术、维护亲密关系 | PEAS 第一步就是定义 P，但这类场景的"成功"本身在动态协商中。强行定义 P 会把一个需要持续对话的关系问题变成"我的 KPI 你打几分"——毁灭性的简化 | 用非暴力沟通（NVC）框架替代 PEAS——先不定义"成功"，先定义"双方的感受和需要" |
| 极反应性危机——火灾、急救、情绪崩溃的现场 | 这时 reflex 反而是**正确架构**。用 PEAS 慢慢拆解效用函数是在浪费逃生窗口 | 事前建立 reflex 级 SOP（火灾 = 跑 / CPR = 按压 / 情绪崩 = 离开现场深呼吸），危机时不思考，执行 SOP |
| 强烈不确定的探索期——你在做一个完全没做过的事，连"什么算成功"都不知道 | Learning Agent 需要稳定反馈来更新部件。探索期的反馈嘈杂且延迟——agent 会学到错误模式（"按了按钮灯亮了但我按的是一个无关按钮"） | 切换为"探索 agent"模式：目标不是最大化效用，而是最大化**信息获取**（每步动作后能减少多少不确定性）。用贝叶斯实验设计思维替代效用最大化 |

### 知识体系定位

PEAS 在 KDO 工具库中的角色：

```
认知升级十步框架（上游方法）
    ↓ 调用
PEAS 智能体分析（本卡 — 中游诊断工具）
    ↓ 诊断结果输入
具体域工具（下游执行）
    ├── 个人修炼：时间管理/深度复盘
    ├── 创业：五步法/精益验证
    └── 管理：科学招聘/目标管理
```

本卡是中游——它不告诉你具体怎么管理时间或怎么招人，它告诉你"你先看清楚自己的 PEAS 和 agent 架构，然后才知道该拿哪个下游工具"。

## Action Triggers

| 触发场景 | 第一个动作 | 成功指标 |
|---------|-----------|---------|
| 接到一个模糊的新任务/新项目（"你负责把这个做了"） | 拿一张纸或新建 .md，画 P/E/A/S 四个格子。**P 格列不满 3 个可量化指标就停下——去找能定义 P 的人。不要继续。** | ≤10 分钟完成四格初填；P 格至少 3 条且可被第三方验证（不是"做好"而是"X 指标达到 Y"） |
| 某件事反复做不好、流程总卡、改了方法也没用 | 别急着改流程。先诊断：你现在用哪种 agent 架构在处理这件事？这个环境真正需要哪种架构？把答案写下来。**如果两者不匹配——你不是做错了，你是选错了架构。** | 5 分钟内写出"当前架构"与"环境需要的架构"；如不匹配，写出 1 个具体动作让你向正确架构靠近一步 |
| 你要做一个不可逆 + 高代价的决定（离职、创业、大额投资） | 写效用函数：U = w1×安全 + w2×成长 + w3×收入 + ... **关键不是算出来的数，是写的过程中你会发现自己根本没想清楚某几个变量的权重。写完念给一个了解你的人听——对方皱眉的地方就是你的盲区。** | 完成效用函数初稿（≥3 个变量，有权重）；至少 1 个变量的权重被 reviewer 质疑并修正；修正后你的下一步动作有明确变化 |
